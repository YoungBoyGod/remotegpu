## 1. 平台介绍

### 1.1 产品概述

VirtAI Cloud (趋动云) 是**业界首个 AI 算力池化云服务**，由趋动科技开发。平台的使命是"连接算力·连接人"，以高性能、高效率、低成本的方式调度全球分布的算力资源。

**核心定位**:
- 为 AI 相关企业、科研院所和个人开发者提供像"水电"一样便宜易用的 AI 算力基础设施服务
- 通过算力池化技术实现资源的统一调度和优化配置

### 1.2 平台优势

#### 成本优势
- 完成相同精度的 ResNet 和 Tacotron2 模型训练，VirtAI Cloud 可为用户节省 **60%-70%** 的成本
- 相比典型公有云 GPU 算力，具有显著的价格优势

#### 易用性
- 提供丰富的预配置环境镜像，免维护的平台服务
- 支持主流开发环境：JupyterLab、VSCode
- 一键零门槛部署分布式训练
- TensorBoard 实验结果可视化

#### 平台能力
- **数据准备**: 提供高质量开放数据集
- **团队协作**: 支持代码、数据、镜像共享
- **算法开发管理**: 结构化的开发流程
- **开发者社区**: 算法资源和预训练模型共享

### 1.3 架构设计

#### 项目组织架构

平台采用**项目为中心**的组织方式：

```
工作空间 (Workspace)
├── 项目 1 (Project)
│   ├── 项目代码 (Code)
│   ├── 开发环境 (Development Environment)
│   ├── 离线训练 (Offline Training)
│   └── 推理服务 (Inference Service)
├── 数据集 (Datasets)
├── 模型库 (Models)
└── 镜像库 (Images)
```

#### 核心组件

**1. 开发环境 (Development Environment)**
- 用途：代码调试空间
- 特点：单机能力，临时数据存储
- 适用场景：测试和代码迭代
- 工作机制：系统根据关联镜像构建环境，挂载数据、模型和项目代码到指定路径

**2. 离线训练 (Offline Training)**
- 用途：大规模模型训练
- 特点：分布式多机多卡训练，持久化结果存储
- 适用场景：生产级模型训练
- 工作机制：从关联镜像创建训练环境，挂载相关资源和代码到适当目录

**3. 推理服务 (Inference Service)**
- 用途：在线预测服务
- 特点：支持服务创建、升级和回滚
- 配置：可配置模型、数据集和镜像

**4. 数据与资源管理**
- 支持预上传的数据和模型
- 提供官方容器镜像库
- 支持自定义镜像创建

### 1.4 工作流程

标准的 AI 开发工作流程：

1. **创建项目** → 基于业务目标建立项目
2. **上传代码** → 将代码上传到项目中
3. **启动开发环境** → 进行代码调试
4. **提交离线训练** → 进行大规模模型训练
5. **创建推理服务** → 部署模型进行在线预测

## 2. 平台计费

### 2.1 计费模式

平台采用**工作空间（Space）维度的按量付费**模式：
- 计费和充值均基于工作空间维度
- 成员任务消耗资源从工作空间余额中按计费规则扣除
- 1 算力点 = 1 人民币

### 2.2 计费项目

平台按资源消耗分 5 大类计费：

#### 1. 开发环境计费
- CPU
- 内存
- GPU
- 显存 (VRAM)
- 临时存储

#### 2. 离线训练计费
- 资源规格费用
- 持久化存储费用

#### 3. 推理服务计费
- 按副本实例收费

#### 4. 可视化任务计费
- 仅 CPU 和内存

#### 5. 数据/代码/模型管理计费
- 仅在存储增加时收费

### 2.3 资源规格定价

**计费公式**:
```
总价 = 资源规格单价 × 使用时长
```

**计费规则**:
- 使用时长：从任务启动到完全停止，向上取整到分钟
- 计费时机：每小时计费一次，任务完成时计费
- 价格调整：仅影响调整日期后新启动的任务
- 跨月任务：为每个日历月生成单独账单

### 2.4 持久化存储定价

**计费公式**:
```
费用 = (日存储平均值 - 免费存储额度) × 单价
```

**计费详情**:
- 单价：0.003333 算力点/GB/天
- 采样频率：每小时采样存储量
- 计算方式：计算日平均值
- 扣费时间：每日午夜进行扣费
- 免费额度：50GB
- 欠费期间：仍会继续计费

**重要提示**:
- 必须主动删除数据才能避免持续计费
- 超过免费存储限额才会产生费用

### 2.5 GPU 资源分类

GPU 资源按算力分为三个版本：

- **B 版本** (Basic): 基础算力
- **S 版本** (Standard): 标准算力
- **P 版本** (Premium): 高级算力

同一等级内，数字越大表示性能越强。

### 2.6 欠费后果

**账户余额耗尽时**:
- 无法启动新的开发环境
- 无法提交新的训练任务
- 无法上传数据
- 运行中的任务在下一个小时边界停止
- 数据在欠费超过免费存储限额后 60 天面临删除风险

### 2.7 充值与发票

**充值方式**:
- 通过平台右上角的 **¥** 符号进入费用中心充值

**查看费用**:
- 费用中心 → 费用账单 → 账单明细
- 支持账单下载

**发票申请**:
- 通过费用中心的"发票管理"(Invoice Management) 部分申请
- 详细流程参见平台发票申请指南

## 3. 快速开始

### 3.1 平台访问

- **文档中心**: https://platform.virtaicloud.com/gemini/v1/gemini_doc/
- **控制台**: https://platform.virtaicloud.com/gemini_web/workspace/space
- **官网**: https://www.virtaicloud.com

### 3.2 新手流程

#### 第一步：创建项目
1. 从平台主页左侧导航栏访问"项目"部分
2. 点击右上角"创建项目"按钮
3. 填写项目信息：
   - **项目名称**: 工作空间内唯一，最多 64 个字符
   - **项目描述**: 简要描述研发目标
4. 确认后选择下一步操作：
   - 取消：不初始化开发环境
   - 先上传代码：推迟启动，先上传代码
   - 立即启动：立即开始开发环境设置

#### 第二步：上传数据
1. 左侧导航栏进入"数据"
2. 点击右上角"创建数据"
3. 填写数据集信息（名称、标签、可见性设置）
4. 选择上传方式：
   - **Web 上传**: 适用于小文件（<2GB，最多 50 个文件）
   - **SFTP 传输**: 适用于大文件（>2GB）

#### 第三步：准备镜像
- 使用官方预配置镜像（推荐新手）
- 或创建自定义镜像

#### 第四步：启动开发环境
1. 进入项目
2. 选择"开发" → "开发环境"
3. 配置资源规格
4. 启动环境

#### 第五步：开发调试
使用三种在线工具：
- **JupyterLab**: 默认打开 `/gemini/code/` 目录
- **Web Terminal**: 具有 root 权限，默认进入 `/gemini/code/` 目录
- **TensorBoard**: 启动命令 `tensorboard --logdir=/gemini/output/`

#### 第六步：提交训练任务
1. 项目 → 训练 → 离线任务
2. 点击"提交任务"
3. 填写任务信息和配置
4. 提交

## 4. 操作指南

### 4.1 项目管理

#### 4.1.1 项目概述

**项目的核心功能**:
- 基于业务目标创建项目
- 上传代码
- 启动开发环境进行调试
- 提交离线训练进行大规模模型训练
- 创建推理服务进行在线预测

**项目优势**:
- 在统一项目内整合研究流程和结果
- 系统化文档管理
- 集中结果存储
- 简化模型部署以进行性能验证

#### 4.1.2 创建项目

**前置条件**: 无

**创建步骤**:
1. 访问项目部分（平台主页左侧导航栏）
2. 点击右上角"创建项目"按钮
3. 在对话框中输入项目详情：
   - 项目名称：工作空间内唯一，不超过 64 个字符
   - 项目描述：简要描述研发目标
4. 点击确认

**创建后选项**:
- **取消**: 退出而不初始化开发环境
- **先上传代码**: 推迟启动，先上传代码
- **立即启动**: 立即开始开发环境设置

**管理功能**:
- **编辑属性**: 通过"..."菜单修改项目名称、描述和标签
- **删除**: 删除项目（注意：从该项目中导出的模型、数据不会被删除）
- **代码上传**: 创建后上传项目特定代码

#### 4.1.3 开发环境与离线训练对比

| 特性 | 开发环境 | 离线训练 |
|------|----------|----------|
| 机器能力 | 单机 | 分布式多机多卡 |
| 数据存储 | 临时存储 | 持久化结果存储 |
| 适用场景 | 测试和代码迭代 | 生产级模型训练 |

### 4.2 数据管理

#### 4.2.1 创建和上传数据集

**文件要求**:
- 文件名：1-64 个字符，不可含 `/`
- 压缩文件格式：tar, zip, tar.gz, rar

**创建流程**:
1. 左侧导航栏进入"数据"
2. 点击右上角"创建数据"
3. 填写数据集信息（名称、标签、可见性设置）
4. 点击"创建"建立空数据集

**上传方法**:

##### Web 上传（适用于轻量数据）
- 支持每次最多 50 个文件
- 单个文件最大 2GB
- 支持上传后调整、解压、删除、重命名
- 保存为版本快照

##### SFTP 传输（适用于大数据集 >2GB）
- 提供 SFTP 工具连接凭证
- 支持工具：cmd, terminal, WinSCP, Xftp
- 传输完成后需手动刷新
- 允许上传后文件管理

**重要提示**:
- 数据上传消耗持久化存储，超过免费存储额度将产生费用
- 每个任务最多可同时挂载 3 个数据集

#### 4.2.2 数据上传下载方法

平台提供多种数据传输方式：

**1. JupyterLab 工具**
- 启动开发环境后，通过内置的 jupyterlab 工具上传下载

**2. SSH 工具**
- 启用 SSH 后，可使用：
  - PyCharm
  - VSCode
  - XFTP
  - WinSCP

**3. 命令行选项**
- 系统支持：git, wget, rm, sftp 命令

**4. 模块集成**
- 平台的数据、模型、代码模块均配有上传功能

**关键限制**:
- **无国际访问**: 平台不支持访问国外网络，国际下载可能超时
- **只读数据集**: 挂载到容器的数据集在开发和离线任务环境中均为只读
- **代码集权限**: 代码集仅在开发环境中允许读写访问

**解决方案**:
- 如果初始未配置 SSH 凭证，可在项目的开发模块中使用"修改 SSH 配置"按钮启用 SSH 功能
- 对于国际源超时问题，推荐先在本地下载，然后上传到平台

### 4.3 模型管理

#### 4.3.1 创建和上传模型

**概述**:
- 平台允许用户创建和上传模型，作为训练任务的预训练模型
- 提高训练效率
- 模型上传消耗持久化存储，超过免费配额可能产生费用

**文件要求**:
- 文件名：1-64 个字符，不可含 `/`，不可单纯为 "." 或 ".."
- 压缩文件格式：tar, zip, tar.gz, rar

**模型集创建流程**:
1. 进入"模型"页面
2. 点击右上角"创建模型"
3. 完成模型信息表单
4. 点击"创建"，然后选择上传方法

**上传方法**:

##### Web 上传
- 最适合小型、轻量数据
- 支持所有文件格式
- 限制：
  - 每次上传最多 50 个文件
  - 单个文件最大 2GB
- 文件操作：解压、删除、重命名

**流程**: 选择 web 上传选项 → 添加文件（一次最多 3 个）→ 根据需要调整文件 → 点击确认保存为版本

##### SFTP 传输
- 适用于超过 2GB 的文件或需要增强安全性
- 用户通过 SFTP 工具连接（cmd, terminal, PowerShell, WinSCP, Xftp）
- 使用提供的连接凭证
- 上传文件 → 关闭连接 → 手动刷新 → 调整并确认文件

两种方法都将上传保存为版本，供训练和调试任务使用。

### 4.4 资源配置

#### 4.4.1 资源类型

**GPU 按算力分类**:
- **B 版本** (Basic): 基础算力
- **S 版本** (Standard): 标准算力
- **P 版本** (Premium): 高级算力

同一等级内，数字越高表示性能越强。

#### 4.4.2 配置原则

**1. 选择流程**
- 先选择资源类型
- 再从可用选项中选择具体规格

**2. 配额管理**
- 任务不能超过个人配额限制或工作空间配额总额
- 配额不足时触发等待队列，直到其他任务释放资源
- 资源不足时同样将任务放入资源队列

**3. 跨云优化提示**
- 优先选择没有跨云指示器的资源，避免数据同步
- 对于大型数据集，选择与数据集云颜色指示器匹配的资源规格
- 对于大型模型，选择与模型云标识符对齐的规格
- 在"项目-数据"部分检查数据/模型同步指示器

#### 4.4.3 配额调整

**工作空间配额增加**:
- 需要联系运营管理员

**个人配额增加**:
- 通过工作空间所有者
- 在空间管理配额设置中管理

### 4.5 开发环境

#### 4.5.1 在线工具

平台提供三种在线开发工具用于调试：

##### JupyterLab
- 默认打开 `/gemini/code/` 目录
- 用户可访问 JupyterLab 文档获取详细信息

##### Web Terminal
- Terminal 用户拥有 root 权限
- 默认进入 `/gemini/code/` 目录
- 平台技术参考部分提供额外的终端使用指导

##### TensorBoard
- 平台启动命令：`tensorboard --logdir=/gemini/output/`
- 用户应根据 TensorBoard 官方文档配置可视化文件输出到 `/gemini/output` 目录

**关键要求**:
- 开发环境必须处于"运行"状态才能访问这些工具
- 所有代码更改必须在 `/gemini/code/` 中进行并保存
- 其他目录是临时的，环境重启后将丢失
- 通过项目开发页面点击"进入开发环境"访问

**重要警告**:
- 关闭网页会终止终端并停止运行任务
- Notebook 休眠、关闭/刷新网页或网络不稳定会导致任务中断和孤立进程消耗资源
- 可能触发内存不足错误
- 长时间运行任务需要持久连接方法，如 JupyterLab 或远程 SSH 客户端

#### 4.5.2 远程开发

##### SSH 连接说明

**概述**:
- 平台通过 SSH 启用到开发环境的远程连接
- 允许本地 IDE 工具和命令行实用程序与云环境交互

**获取 SSH 连接详情**:
1. 进入项目的"开发"部分
2. 访问"项目代码"页面
3. 复制显示的 SSH 信息

**重要提示**:
- 确保开发环境正在运行且 SSH 设置已启用
- 连接详情在环境重启后可能会改变

**SSH 连接字符串分解**:

示例: `ssh -p 30022 gemini@root@ssh-e3eb4194278df0f91258632682d0ae92.default@ssh.virtaicloud.com`

- **Host**: `ssh.virtaicloud.com`
- **Port**: `30022`
- **Username**: `gemini@root@ssh-e3eb4194278df0f91258632682d0ae92.default`
- **Password**: 在平台设置 > SSH 配置中设置

**测试连接**:

- **Mac**: 使用终端 (Cmd+Space) 输入 SSH 命令，如提示验证则确认"yes"
- **Windows 10+**: 打开 CMD (Win+R)，粘贴 SSH 命令，输入密码

**可用连接工具**:
- **PyCharm**: 需要专业版；支持文件传输和代码调试
- **VSCode**: 支持文件操作和调试
- **CMD/PowerShell/Mac Terminal**: 原生远程文件操作工具
- **WinSCP/Xftp**: 图形文件管理器，具有目录操作功能

**故障排除**:

*权限被拒绝*:
- 表示密码错误
- 在 SSH 配置设置中重置凭证
- 注意：连续 5 次失败尝试会锁定账户 30 分钟

*认证失败*:
- 验证用户名/密码准确性
- 检查 `/root/` 和 `.ssh/` 目录权限
- 如果显示 "drwxrwxrwx" (777)，更改为 700：`chmod -R 700 /root/`

##### VSCode 连接开发环境

**概述**:
- 用户可以配置本地 VSCode 远程连接到创建的开发环境
- 实现实时代码同步

**前置条件**:
- 从官方源下载并安装 VSCode

**连接步骤**:

**1. 安装 Remote-SSH 插件**
- 打开 VSCode 的扩展市场
- 搜索 "remote"
- 找到并安装 **Remote-SSH** 扩展
- 等待安装完成

**2. 配置 SSH 设置**
- 在 VSCode 的远程资源管理器中，选择 **+** 按钮
- 输入开发环境的 SSH 连接字符串，按回车
- 选择 SSH 配置文件（通常是 `C:\Users\xxx\.ssh\config`）
- 成功添加后显示确认消息

**3. 连接到环境**

*方法一*:
- 点击右下角通知中的连接按钮
- 输入 SSH 密码（来自凭证管理设置）

*方法二*:
- 点击左侧边栏的刷新按钮

### 4.8 镜像管理

#### 4.8.1 编写 Dockerfile

**关键限制**:

- 推荐 **Ubuntu 20.04** 作为基础操作系统以获得最佳兼容性
- Python 版本必须与操作系统匹配：
  - Ubuntu 18.04 → Python 3.6
  - Ubuntu 20.04 → Python 3.8
  - Ubuntu 22.04 → Python 3.10
- 平台级限制阻止用户手动指定 FROM、EXPOSE、CMD 和 ENTRYPOINT 指令

**基本指令**:

**ENV**: 设置环境变量
- 格式：`ENV [variable] [value]`
- 对后续 RUN 命令可访问

**RUN**: 在容器内执行命令
- 创建新的镜像层
- 通常用于包安装
- 可以使用 `&&` 和 `&& \` 组合多个命令

**包管理方法**:

##### apt (Ubuntu 包管理器)
- 提供来自 Ubuntu 仓库的稳定、系统集成的包
- 默认国外镜像可能不稳定；在 `/etc/apt/sources.list` 中切换到国内源
- 清理缓存：`apt-get clean && rm -rf /var/lib/apt/lists/*`

##### pip (Python 包管理器)
- 访问 PyPI，具有更广泛的包种类和版本选项
- 全局或单次安装配置国内镜像（阿里云、清华、USTC 可用）
- 使用 `--no-cache-dir` 标志减少镜像大小
- 支持版本固定：`pip install package==version`

**镜像优化技术**:
- 合并指令以减少层数
- 使用 `--no-install-recommends` 删除 apt 缓存文件
- 在 pip 安装中添加 `--no-cache-dir`
- 为开发与生产环境采用多阶段构建

**特殊软件注意事项**:
- CUDA、NCCL 和 numpy 在官方镜像中预装
- Deepspeed 需要 libaio 和 triton==1.0.0（与 Python 3.10 不兼容）
- 通过仔细管理依赖关系避免安装 matplotlib、imageio 和 apex 时的版本冲突

**Dockerfile 全局 pip 镜像配置示例**:
```dockerfile
RUN pip config set global.index-url https://mirror.baidu.com/pypi/simple
```

## 6. 最佳实践

### 6.1 任务后台执行

#### 6.1.1 问题概述

**核心问题**:
- 关闭 web 终端或浏览器窗口时，运行的任务会停止
- 因为会话终止导致任务停止
- "关闭终端则任务停止"是常见的行业问题，非平台特定

#### 6.1.2 三种后台运行方法

##### 方法 1: tmux (Terminal Multiplexer)

**概述**: 强大的工具，可同时管理多个终端会话

**安装**:
```bash
sudo apt-get -y install tmux  # Ubuntu
sudo yum install tmux         # CentOS
```

**基本命令**:
- `tmux new -s <name>` - 创建命名会话
- `tmux a -t <name>` - 附加到会话
- `tmux ls` - 列出所有会话
- `Ctrl+b, d` - 从会话分离（保持运行）

**使用案例**:
- "在开发环境提交了一个简单训练，训练过程需要 10 分钟"
- 为不同任务创建单独的会话

##### 方法 2: Screen (命令行窗口管理器)

**概述**: 与 tmux 类似但界面更简单

**关键命令**:
- `screen -S <name>` - 创建会话
- `screen -r <name>` - 恢复会话
- `Ctrl+a+d` - 分离会话
- `screen -ls` - 列出会话

##### 方法 3: nohup (No Hang Up)

**概述**: 最简单的快速后台执行方法

**语法**: `nohup Command [Arg...] &`

**示例**: `nohup /root/runoob.sh &`

**输出**: 成功时返回 "appending output to nohup.out"

#### 6.1.3 替代方案：JupyterLab

平台内置的 JupyterLab 终端即使在关闭 web 浏览器后仍继续运行，支持多个同时终端。

#### 6.1.4 关键警告

**严重问题**:
- "Notebook 休眠、关闭/刷新网页或网络不稳定导致任务中断和孤立进程消耗资源"
- 可能触发内存不足错误

- Ubuntu 22.04 - 兼容性最差，不推荐

#### Q23: Python 版本如何选择？
**A**: 必须与操作系统匹配：
- Ubuntu 18.04 → Python 3.6
- Ubuntu 20.04 → Python 3.8
- Ubuntu 22.04 → Python 3.10

#### Q24: Dockerfile 有哪些限制？
**A**: 平台不允许用户指定：
- FROM（基础镜像由平台指定）
- EXPOSE（端口配置由平台管理）
- CMD（启动命令由平台配置）
- ENTRYPOINT（入口点由平台管理）

### 7.7 推理服务相关

#### Q25: 如何访问推理服务？
**A**:
1. 在推理服务页面添加端口
2. 系统分配外部端口（约 1 分钟）
3. 使用外部地址访问（域名:端口）
4. 注意使用正确的协议（HTTP/HTTPS）

#### Q26: 推理服务访问失败怎么办？
**A**: 检查：
1. 程序是否已正确启动
2. 访问协议是否与程序默认协议匹配
3. 外部端口是否仍然有效（48 小时不活动后释放）

### 7.8 SSH 连接相关

#### Q27: SSH 连接"权限被拒绝"怎么办？
**A**:
- 表示密码错误
- 在"平台设置 > SSH 配置"中重置密码
- 注意：连续 5 次失败会锁定账户 30 分钟

#### Q28: SSH 认证失败怎么办？
**A**:
1. 验证用户名/密码准确性
2. 检查 `/root/` 和 `.ssh/` 目录权限
3. 如果权限是 777，更改为 700：`chmod -R 700 /root/`

#### Q29: VSCode 无法打开 .ipynb 文件怎么办？
**A**: 从 VSCode 市场安装 Jupyter 扩展。

## 9. 技术参考

### 9.1 工具参考

#### 9.1.1 CMD 工具

**概述**:
- CMD (Command Processor) 提供命令行操作
- 用于远程开发环境和平台存储访问

**主要功能**:
- 远程 SSH 访问开发环境
- SFTP 协议上传/下载数据、模型和代码
- 本地机器和开发环境之间的 SCP 传输

**关键操作**:

**打开 CMD**:
- 在搜索框中搜索 "cmd"，按回车

**SSH 连接**:
```bash
ssh -p 30022 gemini@root@ssh-xxx.default@ssh.virtaicloud.com
# 输入平台设置 > SSH 配置中的密码
```

**SFTP 存储连接**:
```bash
# 复制 SFTP 连接字符串
sftp -P port user@host
# 输入提供的密码

# 上传
put -r local_directory /upload

# 下载
get -r /upload local_directory
```

**SCP 文件传输**:
```bash
# 上传
scp -P 30022 -r local_directory dev_address:/gemini/code

# 下载
scp -P 30022 -r dev_address:remote_directory local_directory
# 输入 SSH 密码
```

**远程目录说明**:
- 代码：`/gemini/code/`
- 数据：`/gemini/data-*/`
- 模型：`/gemini/pretrain*/`

#### 9.1.2 Mac 终端

**功能**:
- 与 CMD 类似，提供命令行操作
- 原生 Unix 命令支持

**打开方式**:
- Cmd + Space，输入 "Terminal"

**使用方法**:
- SSH 连接：与 CMD 相同
- SFTP/SCP：与 CMD 相同

#### 9.1.3 Web 终端

**概述**:
- 平台工具，用于执行命令
- 关键限制：关闭网页会终止终端并停止运行任务

**后台执行解决方案**:
1. **tmux/screen 工具** - 终端多路复用器，独立维护会话
2. **nohup 命令** - 语法：`nohup Command [Arg] [&]`，启动不受终端断开影响的进程
3. **离线训练** - 平台原生训练功能，完成时自动终止
4. **JupyterLab 终端** - 内置终端，关闭网页时持续运行

**关键工具**:

**tmux**:
```bash
# 创建会话
tmux new -s name

# 分离
Ctrl+b, d

# 列出会话
tmux ls

# 附加
tmux a -t name
```

**Screen**:
```bash
# 创建
screen -S name

# 重新连接
screen -r name

# 分离
Ctrl+a+d
```

**nohup**:
```bash
nohup /root/script.sh &
```

**关键警告**:
- Notebook 休眠、关闭/刷新网页或网络不稳定导致任务中断
- 孤立进程消耗资源，可能触发 OOM 错误
- 长时间运行任务需要持久连接方法
