# RemoteGPU Backend 项目启动指南

## 📋 项目概述

RemoteGPU Backend 是一个基于 Go + Gin + GORM + Kubernetes 的云GPU管理平台后端系统。

## 👥 团队角色分配

- **项目经理（PM）**: 整体协调、进度管理、代码审查、文档管理
- **开发人员1**: 模块A - 用户认证与权限管理
- **开发人员2**: 模块B - 工作空间与组织管理
- **开发人员3**: 模块C - 基础设施管理（Host、GPU）
- **开发人员4**: 模块D - 环境与容器管理
- **开发人员5**: 模块E - 资源配额与计费
- **开发人员6**: 模块F - 公共基础设施（优先级最高）
- **测试工程师**: 集成测试与质量保证

## 🔄 开发顺序

### 第一阶段（立即开始）
- **任务#7**: 模块F - 公共基础设施（开发人员6）
  - 无依赖，最高优先级
  - 其他所有模块都依赖它

### 第二阶段（模块F完成后）
- **任务#2**: 模块A - 用户认证与权限管理（开发人员1）

### 第三阶段（模块A完成后，可并行）
- **任务#3**: 模块B - 工作空间与组织管理（开发人员2）
- **任务#4**: 模块C - 基础设施管理（开发人员3）

### 第四阶段（模块B完成后）
- **任务#6**: 模块E - 资源配额与计费（开发人员5）

### 第五阶段（所有前置模块完成后）
- **任务#5**: 模块D - 环境与容器管理（开发人员4）

### 第六阶段（所有开发完成后）
- **任务#8**: 集成测试与质量保证（测试工程师）

## 📝 文档规范

### 文档结构
```
docs/
├── modules/          # 各模块设计文档
├── api/              # API接口文档
├── testing/          # 测试文档
```

### 文档要求
1. **每完成一个重要功能后立即更新文档**
2. API变更时同步更新接口文档
3. 记录设计决策和实现细节
4. 记录已知问题和解决方案

## 🔧 Git提交规范

### Commit Message格式
- 模块A: `[ModuleA-User] 功能描述`
- 模块B: `[ModuleB-Workspace] 功能描述`
- 模块C: `[ModuleC-Infrastructure] 功能描述`
- 模块D: `[ModuleD-Environment] 功能描述`
- 模块E: `[ModuleE-Quota] 功能描述`
- 模块F: `[ModuleF-Base] 功能描述`
- 测试: `[Testing] 测试描述`
- 项目管理: `[PM-Doc] 文档描述`

### 提交要求
1. 每完成一个功能点立即提交
2. 提交前确保测试通过
3. 重要功能完成后创建feature分支并提交PR
4. 项目经理审查后合并到主分支

### Commit Message示例
```bash
[ModuleF-Base] 实现数据库连接池管理

- 添加PostgreSQL连接池配置
- 实现连接池健康检查
- 添加单元测试

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

## 🚀 快速开始

### 1. 环境准备
```bash
# 安装依赖
go mod download

# 配置环境变量
cp config/config.example.yaml config/config.yaml

# 启动数据库
docker-compose up -d postgres redis
```

### 2. 开发流程
1. 查看自己的任务: 使用任务管理系统查看任务详情
2. 创建feature分支: `git checkout -b feature/module-x-功能名`
3. 开发功能并编写测试
4. 更新文档
5. 提交代码: 使用规范的commit message
6. 创建PR并请求代码审查

### 3. 测试要求
- 单元测试覆盖率 > 80%
- 提交前运行 `go test ./...`
- 集成测试在CI中自动运行

## 📞 沟通协作

- **每日站会**: 同步进度和问题
- **代码审查**: PR必须经过项目经理审查
- **文档共享**: 所有文档提交到git仓库
- **问题追踪**: 使用任务管理系统记录问题
